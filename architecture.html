VocabMaster 系统架构

Flutter 单词学习 App · 本地 Hive 存储 · 生成于 2026-06-29

概述

VocabMaster 当前是一套 Flutter + Riverpod + Hive 本地存储 的单体 App 架构， 已完全去掉 Isar，所有运行时数据均通过 Hive 读写。

技术栈

层技术
UIFlutter (Material 3)
状态管理flutter_riverpod
本地数据库Hive（5 个 Box）
语音TtsService（系统 TTS）
通知NotificationService
学习算法SM-2（sm2_algorithm.dart）

启动与数据流

Future<void> main() async {
  WidgetsFlutterBinding.ensureInitialized();
  await HiveService.init();
  await HiveService.importInitialBooks();
  await HiveService.seedDefaultSettingsIfNeeded();
  await SessionRepository().cleanupStaleSessions();
  await TtsService.instance.init();
  await NotificationService.instance.init();
  runApp(const ProviderScope(child: VocabMasterApp()));
}

两种学习入口

1. 日常训练（StudyPage）

  • 按每日配额、SM-2 到期词拉取单词
  • 走 studyWordsProvider

2. 关卡测试（LevelSelectionPage → 测试 Tab）

  • 每 30 词一关（level_utils.dart）
  • overrideWords 限定当前关卡词表
  • 会话类型：level_{bookId}_{levelIndex}_challenge_{mode}
  • 满分 100% 获得 1 颗星，3 种模式最多 3 星（level_challenge.dart）

会话编排中心

StudySessionPage 统一负责：

  • 创建 LearningSession
  • 按模式分发到各 Page
  • 三种测试模式共用 QuizSessionResult + quiz_complete_dialog
  • 完成后更新成就、关卡星级

当前架构特点（总结）

  1. 纯本地、离线优先：Hive 为唯一数据源，无后端
  2. Feature 模块化：UI 按业务域拆分，状态集中在 Provider
  3. Repository 模式：UI 不直接碰 Hive，经 Repository 访问
  4. 关卡 + 测试闭环：学习（详情页顺序学）与测试（三模式闯关）共用同一套词书数据
  5. 测试词库驱动：TEST_40（40 词 / 2 关）用于全流程验证
